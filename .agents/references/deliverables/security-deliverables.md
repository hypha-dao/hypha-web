### Security Engineering Deliverables

| Artifact                         | Description                                                                                  |
| -------------------------------- | -------------------------------------------------------------------------------------------- |
| **Threat Model**                 | Diagram and analysis of trust boundaries, data flows, threat actors, and mitigations (STRIDE/PASTA) |
| **Security Review Report**       | Structured findings from code review or architecture review, with severity, evidence, and remediation |
| **Vulnerability Assessment**     | Prioritised inventory of identified weaknesses mapped to CWE/OWASP, with risk ratings and fix guidance |
| **Security Checklist**           | Pre-deployment verification list covering auth, input validation, headers, secrets, dependencies |
| **Remediation Guide**            | Step-by-step fix instructions per finding, including code examples, configuration changes, and verification steps |
| **Security Architecture Review** | Evaluation of system design against defence-in-depth principles, trust boundaries, and data classification |
| **Dependency Audit Report**      | Analysis of direct and transitive dependencies for known CVEs, maintenance status, and license risks |
| **Incident Postmortem**          | Root cause analysis of a security incident with timeline, impact, remediation, and preventive measures |
| **Security Policy Document**     | Organisational or project-level security requirements, standards, and compliance mapping      |
