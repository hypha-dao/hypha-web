### Requirements Engineering Deliverables

| Artifact                       | Description                                                                |
| ------------------------------ | -------------------------------------------------------------------------- |
| **Requirement Specification**  | Structured document with FR/NFR items, acceptance criteria, and context    |
| **Traceability Matrix**        | Bidirectional mapping from goals → requirements → tickets → test cases     |
| **Stakeholder Analysis**       | Identification of stakeholders, their concerns, influence, and engagement  |
| **Glossary / Domain Model**    | Shared vocabulary and conceptual model for the problem domain              |
| **Use Cases / User Stories**   | Behavioral descriptions with actors, flows, and acceptance criteria        |
| **Non-functional Spec**        | Performance budgets, security constraints, accessibility standards         |
| **Impact Analysis**            | Assessment of change requests on scope, timeline, and existing requirements|
| **Kanban Board**               | Visual tracking of features and tickets through the requirements lifecycle |
| **Open Questions Log**         | Tracked list of unresolved items with owners and target resolution dates   |
