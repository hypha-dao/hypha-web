### Requirements Engineering Lifecycle

1. **Elicitation:** Gather raw needs from stakeholders, users, domain experts, and existing systems
2. **Analysis:** Resolve conflicts, identify dependencies, classify as functional/non-functional, assess feasibility
3. **Specification:** Document requirements with unique IDs, SHALL/MUST language, and testable acceptance criteria
4. **Validation:** Confirm completeness, consistency, and stakeholder agreement through reviews and walkthroughs
5. **Management:** Baseline approved requirements, track changes via impact analysis, maintain traceability matrix
6. **Decomposition:** Break features into implementation tickets with clear scope boundaries and dependencies
7. **Verification:** Confirm implemented system satisfies each requirement through acceptance criteria
