### Security Operations Lifecycle

#### 1. Prevention & Hardening
- Establish security baselines and hardening guides
- Deploy preventive controls (WAF, IDS/IPS, endpoint protection)
- Implement patch management and vulnerability remediation cycles
- Configure security monitoring and log aggregation

#### 2. Detection & Monitoring
- Deploy SIEM for centralised log analysis and correlation
- Implement UEBA for anomaly detection
- Configure alerting for authentication failures, privilege escalation, data exfiltration patterns
- Maintain threat intelligence feeds and IOC databases
- Run continuous vulnerability scanning (infrastructure + application)

#### 3. Incident Response
- **Preparation:** Maintain IR playbooks, communication plans, and escalation procedures
- **Identification:** Triage alerts, confirm incidents, assess scope and severity
- **Containment:** Isolate affected systems, block attack vectors, preserve evidence
- **Eradication:** Remove threat actor presence, patch exploited vulnerabilities
- **Recovery:** Restore from verified backups, validate system integrity, resume operations
- **Lessons Learned:** Conduct post-incident review, update playbooks, implement preventive measures

#### 4. Continuous Improvement
- Track security KPIs: MTTD, MTTR, vulnerability density, patch cadence
- Conduct regular tabletop exercises and purple team engagements
- Review and update security policies quarterly
- Benchmark against NIST CSF maturity levels and peer organisations
- Incorporate threat landscape changes into risk assessments
