### Development Lifecycle

#### Feature Development Flow

1. **Specification:** Understand requirements, acceptance criteria, and affected domains (spaces, governance, treasury, members)
2. **Design:** Identify package boundaries — which packages need changes (`core`, `epics`, `ui`, `storage-postgres`, `storage-evm`)
3. **Implementation:** Work bottom-up: schema -> server mutations/queries -> server actions -> hooks -> UI components
4. **Testing:** Unit tests for logic, E2E tests for user flows, contract tests for on-chain behavior
5. **Review:** PR with clear description, schema diff (auto-generated by CI), type safety verified by build
6. **Deploy:** Preview deployment per PR (Vercel + Neon branch), production on merge to `main`

#### Branch Strategy

- `main` is production — protected, deploys automatically
- Feature branches create preview environments with isolated Neon DB branches
- Schema migrations run automatically in CI before deployment

#### Change Propagation

When changing shared types or schemas:

1. Update `storage-postgres` schema and generate migration
2. Update `core` types and server queries/mutations
3. Update `epics` hooks and components
4. Verify build passes across all packages (`turbo build`)
