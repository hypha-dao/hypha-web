### Security Frameworks

| Framework | Purpose | Application |
|-----------|---------|-------------|
| **NIST CSF 2.0** | Cybersecurity risk management across 6 functions: Govern, Identify, Protect, Detect, Respond, Recover | Organisation-wide security programme structure and maturity assessment |
| **NIST SP 800-53 Rev 5** | Comprehensive security and privacy control catalog | Selecting and implementing specific security controls mapped to risk |
| **NIST SSDF (SP 800-218)** | Secure software development practices | Embedding security into SDLC: prepare, protect, produce, respond |
| **NIST SP 800-207** | Zero Trust Architecture reference | Designing and implementing ZTA principles and patterns |
| **OWASP Top 10** | Top web application security risks | Prioritising application security testing and remediation |
| **OWASP SAMM** | Software Assurance Maturity Model | Assessing and improving application security programme maturity |
| **OWASP ASVS** | Application Security Verification Standard | Detailed verification requirements for application security controls |
| **MITRE ATT&CK** | Adversary tactics, techniques, and procedures | Threat intelligence, detection engineering, and red team planning |
| **MITRE ATLAS** | Adversarial ML threat landscape | AI/ML-specific threat modeling and defence |
| **STRIDE** | Threat modeling framework | Systematic threat identification during design reviews |
| **ISO 27001/27002** | Information security management system | Establishing and certifying an ISMS with controls |
| **SOC 2 (TSC)** | Trust Services Criteria for service organisations | Demonstrating security, availability, confidentiality controls to customers |
| **CIS Controls v8** | Prioritised set of cybersecurity actions | Implementing foundational, operational, and organisational security controls |
| **SLSA** | Supply-chain Levels for Software Artifacts | Software supply chain integrity and provenance verification |
