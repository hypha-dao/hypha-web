### Nielsen's 10 Usability Heuristics

Jakob Nielsen's heuristics for interaction design evaluation (Nielsen, 1994). Use these as a systematic lens for heuristic evaluation and design review.

| #  | Heuristic                                              | Core Principle                                                                                   |
| -- | ------------------------------------------------------ | ------------------------------------------------------------------------------------------------ |
| 1  | **Visibility of System Status**                        | Keep users informed about what is going on through timely, appropriate feedback                   |
| 2  | **Match Between System and Real World**                | Use language, concepts, and conventions familiar to users; follow real-world logic                |
| 3  | **User Control and Freedom**                           | Provide clearly marked exits, undo, and redo so users feel in control                            |
| 4  | **Consistency and Standards**                          | Follow platform and industry conventions; maintain internal and external consistency              |
| 5  | **Error Prevention**                                   | Eliminate error-prone conditions or present confirmation before committing to actions             |
| 6  | **Recognition Rather Than Recall**                     | Minimize memory load by making elements, actions, and options visible and retrievable             |
| 7  | **Flexibility and Efficiency of Use**                  | Provide accelerators for experts while keeping the interface usable for novices                   |
| 8  | **Aesthetic and Minimalist Design**                    | Keep content and visual design focused on essentials; remove irrelevant or rarely needed elements |
| 9  | **Help Users Recognize, Diagnose, and Recover from Errors** | Use plain language for errors, indicate the problem precisely, and suggest a solution       |
| 10 | **Help and Documentation**                             | Provide concise, task-focused documentation that is easy to search and contextually relevant      |

#### Severity Rating Scale

| Rating | Label        | Description                                                    |
| ------ | ------------ | -------------------------------------------------------------- |
| 0      | Not a problem | No usability issue identified                                 |
| 1      | Cosmetic     | Fix only if extra time is available                            |
| 2      | Minor        | Low priority; causes minor friction                            |
| 3      | Major        | High priority; significant impact on task completion           |
| 4      | Catastrophic | Must fix before release; prevents users from completing tasks  |

_Reference: Nielsen, J. (1994). "Heuristic evaluation." In Nielsen, J., and Mack, R.L. (Eds.), Usability Inspection Methods. John Wiley & Sons._
