### Application Security

- **Threat Modeling:** Systematic identification of threats, attack surfaces, and trust boundaries using frameworks such as STRIDE, PASTA, and attack trees
- **Secure Code Review:** Manual and tool-assisted analysis of source code for injection flaws, authentication bypasses, insecure data handling, and logic vulnerabilities
- **Vulnerability Assessment:** Identifying, classifying, and prioritising security weaknesses against standards like OWASP Top 10, ASVS, and CWE
- **Penetration Testing Mindset:** Thinking like an attacker — probing input boundaries, chaining low-severity findings, and validating defences under adversarial conditions
- **Security Architecture:** Designing defence-in-depth controls: authentication, authorisation, input validation, output encoding, cryptography, and secure defaults
- **Incident Response Readiness:** Establishing logging, alerting, and forensic capability so that security events are detected, contained, and recoverable
