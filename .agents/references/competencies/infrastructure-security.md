### Infrastructure Security

- **Cloud Security Posture Management:** Configuring and auditing security controls across AWS, GCP, and Azure — IAM policies, VPC design, security groups, encryption at rest/in transit, and cloud-native security tooling (GuardDuty, Security Hub, Cloud Armor)
- **Zero Trust Architecture:** Designing and implementing zero trust principles — verify every request, enforce least-privilege access, micro-segment networks, and assume breach. Aligning with NIST SP 800-207 guidance
- **Identity & Access Management:** Designing RBAC/ABAC/ReBAC models, enforcing MFA, managing service accounts, implementing just-in-time access, and auditing permission sprawl across systems
- **Container & Runtime Security:** Securing container images (base image selection, vulnerability scanning, minimal attack surface), runtime protection, Kubernetes RBAC, network policies, and pod security standards
- **Infrastructure as Code Security:** Scanning Terraform, CloudFormation, and Helm charts for misconfigurations (Checkov, tfsec, KICS). Enforcing security policies as code in deployment pipelines
- **Network Security:** Designing network segmentation, configuring firewalls and WAFs, implementing DDoS protection, managing VPN/mTLS, and monitoring network traffic for anomalies
- **Secrets Management:** Architecting centralised secrets management (HashiCorp Vault, AWS Secrets Manager, Doppler), enforcing rotation policies, preventing secret sprawl, and auditing access to sensitive credentials
- **Encryption & Key Management:** Implementing encryption at rest (AES-256) and in transit (TLS 1.3), managing PKI infrastructure, key rotation schedules, and certificate lifecycle management
