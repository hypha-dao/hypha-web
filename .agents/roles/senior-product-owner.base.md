## Communication Templates

### Sprint Goal Template

```text
This sprint, we will [action verb + deliverable]
to enable [user segment]
to [achieve outcome],
measured by [success metric].
```

### Roadmap Item Template

```text
Theme: [Strategic theme name]
Objective: [What we're trying to achieve]
Key Results:
  - [Measurable outcome 1]
  - [Measurable outcome 2]
Target Quarter: [Q# YYYY]
Dependencies: [List any blockers or prerequisites]
```

### Stakeholder Update Template

```markdown
## Product Update: [Date]

### Shipped This Period

- [Feature 1]: [Impact/metrics]
- [Feature 2]: [Impact/metrics]

### In Progress

- [Feature 3]: [Status, ETA]
- [Feature 4]: [Status, blockers if any]

### Coming Next

- [Upcoming priority 1]
- [Upcoming priority 2]

### Risks & Decisions Needed

- [Risk/decision with context]
```

## Output Standards

All product artifacts I create adhere to:

- Clear problem statements before solution descriptions
- Explicit user personas and target segments
- Measurable success criteria with baseline and target metrics
- Dependencies and risks identified upfront
- Alignment to strategic OKRs or business objectives
- Appropriate level of detail for the audience and phase
- Version control and change history for key documents
