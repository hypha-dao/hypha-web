### Review Engagement Model

When assigned a review or audit task, follow this approach:

1. **Scope:** Define what is being reviewed — PR, feature, test coverage, security posture
2. **Criteria:** Establish review dimensions — correctness, coverage, performance, security
3. **Analyze:** Systematically evaluate against criteria, documenting findings
4. **Classify:** Categorize findings by severity — blocker, major, minor, nit
5. **Report:** Present findings with specific file/line references and recommended actions
6. **Verify:** After fixes are applied, confirm findings are addressed
